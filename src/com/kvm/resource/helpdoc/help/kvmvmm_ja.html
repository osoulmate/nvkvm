Javaリモートバーチャルコンソール

機能概要

Javaリモートバーチャルコンソールが提供する機能を使用して、サーバーにリモートで接続することでサーバーへのリモート制御・管理、オペレーティングシステムのインストール・修復、デバイスドライバーのインストールなどの操作を行うことができます。

  • ローカルPCから遠隔地にあるサーバーをキーボードとマウスを使用してリアルタイムにリモート操作することができます。
  • サーバーからインタネット経由で仮想フロッピードライブや光学ドライブを使用してローカルPCにリモートアクセスすることができます。サーバー側では、仮想フロッピードライブや光学ドライブの使用方法はサーバーに挿入している（USB、Universal Serial Bus）デバイスのと同じです。
    注記:

    ローカルのフロッピードライブや光学ドライブをローカルPCのメディアとして扱えるし、ローカルPCまたはネットワークドライブにあるフロッピーディスクや光ディスクに保存されているイメージファイルをローカルPCのメディアとして扱えます。

[キーボード組み合せキー]ボタン。特殊の組み合せキーを送信またはカスタマイズすることを示します。このウィンドウの組み合せキー及びその意味は以下のとおりです。

  • Ctrl+Shift：入力方式を切り替えます。
  • Ctrl+Esc：[開始]メニューを表示または非表示します。
  • Ctrl+Alt+Del：オペレーティングシステム画面のロック、ユーザーのログアウト、パスワードの変更、タスクマネージャーの起動、サーバーの再起動などの操作が実行できます。
  • Alt+Tab：開いた項目の間で切り替えます。
  • Ctrl+Space：入力方式を有効または無効にします。
  • ResetKeyboad：キーボード上のキーを復旧することをシミュレートします。

iBMC 2.98及びその以降のバージョンのみはクライアントのキーボードレイアウトの選択をサポートします。

作業手順

特別な組み合せキーの送信

  1. KVM画面からツールバーにあるをクリックします。

    組み合せキーウィンドウが表示されます。このウィンドウにある組み合せキー及びその意味は次の通りです。

    • Ctrl+Shift：入力言語の切り替え。
    • Ctrl+Esc：[スタート]メニューを表示または非表示します。
    • Ctrl+Alt+Del：オペレーティングシステム画面のロック、ユーザーのログオフ、パスワードの変更とタスクマネジメントの起動、サーバーの再起動など。
    • Alt+Tab：開いているアイテム内の切り替え。
    • Ctrl+Space：入力言語の有効化または無効化。
    • ResetKeyboad：キーボードの入力のシミュレーション。
  2. 表 1に記載されているパラメーター情報によって、送信する組み合せキーをクリックします。

    サーバーは組み合せキーに対応する操作を実行します。

注記:

組み合わせキーをカスタマイズする場合、[カスタマイズ]後ろにあるテキストボックスにキーを順次入力してから[送信]をクリックします。

クライアントのキーボードレイアウトの指定

注記:

iBMC 2.98及びその以降のバージョンのみはクライアントのキーボードレイアウトの選択をサポートします。

KVM画面に、ツールバーのをクリックします。

プルダウンリストからターゲットのキーボードレイアウトを選択します。キーボードレイアウトを強制的に指定することが成功します。

仮想光学ドライブのマウント

この操作ではローカルPCにある光ディスクドライブを使用して、サーバーに仮想光ディスクドライブを作成して提供します。

  1. KVM画面からツールバーにあるをクリックします。

    図 3に示すような画面が表示されます。

    図 3 仮想光学ドライブのマウント

  2. [光学ドライブ]ラジオボタンを選択します。
  3. プルダウンリストからローカルPCにある仮想化する光ディスクドライブ、例えば、[G:]を選択します。
  4. [接続]をクリックします。

    仮想光学ドライブがサーバーにマウントされています。

    注記:

    マウントに成功した場合、[切断]をクリックし、表示されているオプションを1つ選択するダイアログボックスから[はい]をクリックして、サーバーにある仮想光学ドライブをアンインストールします。

仮想光学ドライブによるイメージファイルのマウント

この操作ではローカルPCにある光ディスクのイメージファイルを使用して、サーバーに仮想光学ドライブを作成して提供し、また、光ディスクのイメージファイルを当該仮想光学ドライブにロードします。

  1. KVM画面からツールバーにあるをクリックします。

    図 3に示すような画面が表示されます。

  2. [イメージファイル]ラジオボタンを選択します。
  3. [閲覧]をクリックします。

    開くウィンドウが表示されます。

  4. ローカルPCに保存されている光ディスクのイメージファイルを選択して[開く]をクリックします。

    図 3に示すような画面に戻します。

  5. [接続]をクリックします。

    イメージファイルがサーバーにマウントされています。

    注記:

    • イメージファイルをマウントすることに成功した場合、[イジェクト]をクリックすると、光ディスクのイメージファイルが表示されます。光ディスクのイメージファイルが表示されている場合、他の*.iso形式のイメージファイルを再選択し、[挿入]をクリックして、当該光ディスクのイメージファイルをロードします。

    • イメージファイルをマウントすることに成功した場合、[切断]をクリックし、表示されているオプションを1つ選択するダイアログボックスから[はい]をクリックして、サーバーにある仮想光学ドライブをアンインストールします。

仮想フロッピードライブのマウント

この操作では、ローカルPCにあるフロッピードライブまたはフロッピーディスクのイメージファイルを使用して、サーバーに仮想フロッピードライブを作成して提供します。

  1. KVM画面からツールバーにあるをクリックします。

    図 4に示すような画面が表示されます。

    図 4 仮想フロッピードライブのマウント

  2. [フロッピードライブ]ラジオボタンを選択します。
  3. プルダウンリストからローカルPCにある仮想化するフロッピーディスクドライブ、例えば、[A:]を選択します。
  4. [書き込み保護]のチェックボックスに チェックを入れます。
    注記:

    書き込み保護はフロッピードライブへのデータ書き込みを禁止します。書き込み保護は重要データの上書き、削除を防止する保護仕組みの1つです。

  5. [接続]をクリックします。

    サーバーに仮想フロッピードライブをマウントすることに成功します。

    注記:

    マウントに成功した場合、[切断]をクリックし、表示されているオプションを1つ選択するダイアログボックスから[はい]をクリックして、サーバーにある仮想フロッピードライブをアンインストールします。

仮想フロッピードライブによるイメージファイルのマウント

この操作では、ローカルPCにあるフロッピーディスクのイメージファイルを使用して、サーバーに仮想フロッピードライブを作成して提供し、また、フロッピーディスクのイメージファイルを該当の仮想フロッピードライブにロードします。

  1. KVM画面からツールバーにあるをクリックします。

    図 4に示すような画面が表示されます。

  2. [イメージファイル]ラジオボタンを選択します。
  3. [閲覧]をクリックします。

    開くウィンドウが表示されます。

  4. ローカルPCに保存されているフロッピーディスクのイメージファイルを選択して[開く]をクリックします。

    図 4に示すような画面に戻します。

  5. [接続]をクリックします。

    仮想フロッピードライブがサーバーにマウントされています。

    注記:

    • イメージファイルのロードに成功した場合、[イジェクト]をクリックすると、フロッピーディスクのイメージファイルが表示されます。フロッピーディスクのイメージファイルが表示されている場合、他の*.img形式のイメージファイルを再選択し、[挿入]をクリックして、当該イメージファイルをマウントします。

    • [切断]をクリックし、表示されているオプションを1つ選択するダイアログボックスから[はい]をクリックして、サーバーにある仮想フロッピードライブをアンインストールします。

イメージファイルの作成

この操作では、フロッピードライブまたは光学ドライブにあるフロッピーディスクや光ディスクを使用してイメージファイルを作成します。作成されたイメージファイルをローカルPCに保存します。仮想フロッピードライブまたは光学ドライブをマウント及びロードするために使用されます。

この操作を実行する前に、ローカルPCのフロッピードライブまたは光学ドライブにフロッピーディスクまたは光ディスクが挿入されていることを確認してください。

  1. KVM画面からツールバーにあるをクリックします。

    図 5に示すような画面が表示されます。

    図 5 イメージファイルの作成

  2. [ドライブの選択]プルダウンリストからクライアントにあるフロッピードライブまたは光ディスクドライブを選択します。
  3. [閲覧]をクリックします。保存ウィンドウが表示されます。
  4. イメージファイルがPCに保存されるパスを選択し、[ファイル名：]テキストボックスにイメージファイル名を入力します。
    注記:

    システムがサポートしているのは形式が*.isoである光ディスクのイメージファイル及び形式が*.imgであるフロッピーディスクのイメージファイルのみです。

  5. [保存]をクリックします。

    図 5に示すような画面に戻します。

  6. [作成]をクリックします。

    作成が完了したら、システムから「イメージファイルの作成に成功しました。」ウィンドウが表示されます。

    [作成の進捗]バーにイメージファイルの作成の進み具合をパーセンテージで表示します。

    注記:

    作成中に、[停止]をクリックすると、イメージファイルの作成を停止することができます。

仮想フォルダーのマウント

この操作ではローカルPCのフォルダーをサーバーにマウントし、サーバーのシステムが読み取り専用の方式でローカルフォルダーにアクセスできるようになります。

注意事項:

仮想フォルダーをマウントする前に、まず転送するファイルをタッゲートフォルダーにコピーしておいてください。仮想フォルダーをマウントした場合、ファイルの追加または削除の操作はできません。

  1. Remote Control画面からツールバーにあるをクリックします。

    図 6に示すような画面が表示されます。

    図 6 仮想フォルダーのマウント
  2. [ローカルフォルダー]ラジオボタンを選択します。
  3. [閲覧]をクリックします。

    ローカルフォルダー選択ウィンドウが表示されます。

  4. マウントするローカルフォルダーを選択して[開く]をクリックします。
  5. [接続]をクリックします。
    注記:
    • 接続が成功した場合、サーバーのオペレーティングシステムから仮想フォルダーを見ることができます。このフォルダーから直接ファイルをコピーすることができます。
    • 接続が成功した場合、[切断]をクリックすると、仮想フォルダーをアンインストールすることができます。

サーバーの電源投入

  1. KVM画面からツールバーにあるをクリックし、ショートカットメニューから[電源ON]を選択します。

    オプションを1つ選択するダイアログボックスが表示されます。

  2. [はい]をクリックします。

    サーバーは電源を投入します。

    注記:

    サーバーの電源投入時間はサーバーの設定によって違います。

サーバーの電源切断

注意事項:

  • 電源をオフにする前に、現在の業務を中断するリスクがないことを確認してください。

  • 実際の状況によって電源切断の方法を選択してください。[強制電源停止]及び[電源OFF]の違いについては、iBMCユーザーガイドの電源管理を参照してください。

  1. KVM画面からツールバーにあるをクリックして、ショートカットメニューから[強制電源停止]または[電源OFF]を選択します。

    オプションを1つ選択するダイアログボックスが表示されます。

  2. [はい]をクリックします。

    サーバーは電源を切断します。

強制再起動または電源を強制的にオフにしてからオンにする

注意事項:

  • 強制再起動または電源を強制的にオフにしてからオンにすると、ユーザーのプログラムや保存されていないデータを壊す可能性があります。そのために、オペレーティングシステムの実際状況によって操作方法を慎重に選択してください。

  • 現在の業務が中断するリスクがないことを事前に確かめてから[強制再起動または電源を強制的にオフにしてからオンにする]を行ってください。

  • 実際の状況によって[システム強制リセット]または[強制電源サイクル]を選択してください。[システム強制リセット]及び[強制電源サイクル]の違いについては[電力] > [電源管理]を参照してください。

  1. KVM画面からツールバーにあるをクリックして、ショートカットメニューから[システム強制リセット]または[強制電源サイクル]を選択します。

    オプションを1つ選択するダイアログボックスが表示されます。

  2. [はい]をクリックします。

    サーバーは[システム強制リセットまたは強制電源サイクル]を行います。

    注記:

    サーバーのシステム強制リセットまたは強制電源サイクルに要する時間は、サーバーの設定によって違います。

キーボードとマウスのリセット

この操作ではUSBキーボード及びUSBマウスの抜き差しをシミュレートします。

  1. KVM画面からツールバーにあるをクリックしてショートカットメニューから[キーボードとマウスのリセット]を選択します。

    オプションを1つ選択するダイアログボックスが表示されます。

  2. [はい]をクリックします。

    サーバーはUSBのリセット操作を実行します。

リアルタイムデスクトップの録画

この操作では、現在、リモートバーチャルコンソールに表示されている画面を動画として録画します。

  1. KVM画面からツールバーにあるをクリックします。

    オプションを1つ選択するダイアログボックスが表示されます。

  2. [はい]をクリックします。

    保存ウィンドウが表示されます。

  3. 録画する動画ファイルがPCに保存されるパスを選択し、[ファイル名：]テキストボックスに動画ファイルの名称を入力します。
  4. [保存]をクリックします。

    KVM画面に戻して動画の録画を開始します。

  5. 録画が完了した後、をクリックします。

    オプションを1つ選択するダイアログボックスが表示されます。

  6. [はい]をクリックします。

    動画ファイルが指定のパスに保存されています。

録画した動画ファイルの形式は*.repです。[録画再生]画面でビデオファイルを再生することやビデオに対するキャプチャーができます。

シングルマウスの使用

ローカルPCのマウスとリアルタイムのマウスが同期しない場合、シングルマウスの機能を使用してローカルPCのマウスを非表示することができます。KVM画面にはリアルタイムのマウスしか表示されていません。

  1. KVM画面からツールバーにあるをクリックしてショートカットメニューから[シングルマウス]を選択します。

    オプションを1つ選択するダイアログボックスが表示されます。

  2. [はい]をクリックします。

    KVM画面にはリアルタイムのマウスしか表示されていません。

リモートマウスの加速

この操作では、リアルタイムデスクトップにあるマウスの速度を速くして、ローカルPCのマウスと同期できるようにします。

  1. KVM画面からツールバーにあるをクリックしてショートカットメニューから[マウスの加速]を選択します。

    オプションを1つ選択するダイアログボックスが表示されます。

  2. [はい]をクリックします。

    ローカルPCとサーバーのマウスを同期化します。
